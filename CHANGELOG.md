# Changelog

All notable changes to `weebly-to-firebase`. Format loosely follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/); the project uses
[semver](https://semver.org/) but is pre-1.0 (breaking changes possible on
minor bumps).

## [0.4.0] — 2026-06-11

The wget-first release. The crawled mirror is now the source of truth; the
Weebly theme export is an optional overlay. End-to-end pipeline runs from
one command; output is buildable without a `WeeblyExport/` present.

### Added
- `crawl` command — `wget --mirror` of the live site, seeded from
  `/sitemap.xml` so JS-rendered nav doesn't drop pages
- `port` command — extracts `<main>` + partials from the crawled mirror
  into `src/html/`, harvests `@font-face` blocks, dumps same-origin
  stylesheets as `src/less/_w2f-*.less`, downloads referenced images to
  `public/assets/gfx/`, rewrites URLs
- `port --all` — port every HTML file in the mirror, scaffold skeletons
  for unknown pages, single command for the bulk first pass
- `forms` command — finalize the contact-form scaffold (`npm install` in
  `functions/`, real hCaptcha sitekey, `firebase functions:secrets:set
  HCAPTCHA_SECRET`); TTY-aware, auto-skips the interactive secret step
  when stdin isn't a terminal
- Auto-pipeline: `w2f init` runs scaffold → crawl → port-all → convert →
  git init in one go, with `--skip-*` opt-outs per step
- Optional `--setup-firebase` — creates the Firebase project + hosting
  site via the Firebase CLI
- Weebly chrome deny-list (`lib/weebly-chrome.mjs`) — Fancybox /
  social-icons / commerce / VideoJS / Select2 stylesheets and the
  matching sprite assets are skipped at port time
- Reusable opt-in modules scaffolded by `init`: `email-hider.js`,
  `embed-consent.{js,less}`, `lightbox.{js,less}` — unused unless wired
  up, each ships with an `@docs` sibling MD
- Opt-in GitHub Actions deploy workflow scaffolded when both
  `--firebase-project` and `--github-repo` are set
- Mobile drawer + hamburger toggle JS port
- JS-driven active-nav marker; body class port
- Open-source scaffolding: LICENSE, CONTRIBUTING, CODE_OF_CONDUCT,
  issue + PR templates, SECURITY policy, npm-publishable metadata

### Changed
- `WeeblyExport/` is dropped into `reference/` from the start, not `src/`
- Live domain normalization shared across crawl / port / convert
- Skeleton partials extracted on first port; subsequent ports leave them
  alone unless `--force`
- `src/less/main.less` import order regenerated by `port` (gated on a
  `Generated by w2f` marker)
- Banner / main wrapper extraction preserves layout divs
- Empty `<span>` / `<div>` preserved for layout fidelity

### Fixed
- Scaffolded README refers to the `w2f` CLI, not the removed `scripts/`
- `@font-face` blocks from linked stylesheets are harvested correctly
- Underline width on active nav
